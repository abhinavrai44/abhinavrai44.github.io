Disparity Coputation using Stereo Cameras

  • by:Abhinav Rai and Satyarth Rraveen

The most standard method for computing disparity map is to use block matching. Basically, we take a small region of pixels in right image and search for the closest match in the left image. The differece in the position of the two regions is the disparity. Regions with large disparity are closer to the camera, while regions with small disparity are farther away from the camera.

Using Stereo Cameras allows us to to view objects in the overlapping field of view of the two cameras from different locations, thus providing different perceptives of the scene. Disparity can be computed by finding the corresponding match of each pixel in the other image. The distance between the pixels location of a feature in one image to its location in other image, can be used to compute the disparity map. This disparity map can then be used to generate a depth map using a simple formula.

$$depth = {(baseline * FocalLength) \over disparity}$$

Because we will only search horizontally, it is important to rectify the images. Sum of Absolute Diffrence (SAD) is computed over the rectified grayscale images as a common similarity metric. Since disparity needs to be computer over every pixel, the biggest challenge is computing it in real time. To achieve this task we need to write highly efficient code which makes use of parallel operations over GPU or SIMD instructions on CPUs.
